# 📊 メイン業務

## 01 📋 在庫管理
**機能**: 在庫データの閲覧・編集
- 商品コード / 統合タイトル / 物流ルート / 補正倍率 を編集して一括保存
- ステータス別（🔴危険 / 🟠要発注 / 🟣過剰）の集計表示
- キーワード検索 + ステータス・物流ルートでフィルタ

**読み**: 04_在庫管理 ／ **書き**: 04_在庫管理（編集可列のみ）

⚠ F/H/K/N/O/P 列は数式列で編集不可。D/E/G/I/J/L/M/R/S/T 列はGAS/バッチ自動書込列で編集不可

## 02 💰 売上管理
**機能**: 売上データ閲覧・編集・CSV取込（3タブ）
- 「📊 閲覧」「✏️ 編集・削除」「📥 CSV取り込み」
- 日付範囲・モール・商品コードで絞込
- 全部値（数式なし）方式

**読み**: 05_売上管理 ／ **書き**: 05_売上管理

⚠ 12,000行超は重い。編集時は必ず期間絞り。

## 03 📦 商品マスタ
**機能**: 商品マスタ閲覧・編集・紐づけ・A列同期
- 「📋 全列」/「🔗 紐づけ専用」ビュー切替
- AE(楽天SKU)→AF(FBM)→AG(FBA) 優先順でA列を自動同期
- チャネル整合性チェック
- サイドカード: 編集中SKUの利益額・利益率をリアルタイム表示

**読み**: 03_商品マスタ参照 ／ **書き**: 03_商品マスタ参照、04_在庫管理、PUSH対象、17_終売SKU（A列連動更新時）

⚠ AC列（手動原価）上書き禁止 / L・M列は数式 / AD列は当月販売数の数式

# 🚀 PUSH

## 04 📤 在庫PUSH
**機能**: 楽天/Amazon FBMへ在庫リアルタイム同期（3タブ）
- 「🏪 楽天」「📦 Amazon FBM」「🚀 一括PUSH」
- F列(自社倉庫) - マスタZ列(バッファ) = PUSH数量
- コバリ子は親在庫÷係数
- チェックボックスで送信SKU選別（チェック0件=全件）

**読み**: 04_在庫管理(F列), 03_商品マスタ参照(AE等), 05_売上管理 ／ **書き**: なし（外部API送信）

⚠ AE列空なら楽天SKU=商品コード扱い

# 📈 集計

## 05 📈 月次サマリ
**機能**: 月×チャネル別の売上・利益・広告費集計
- 過去3〜24ヶ月選択で月別集計
- グラフ: 月別売上推移 / 月別最終粗利推移
- 広告費・アフィリ・ポイント・クーポンを最終粗利から減算

**読み**: 05_売上管理, 広告費入力 ／ **書き**: なし

## 06 📊 大分類別販売集計
**機能**: 大分類×月×チャネル群の3次元集計
- 「💰 売上」「📦 数量」「💎 利益」「📈 グラフ」タブ
- 年・月・チャネル群（楽天/Amazon/全て）選択
- Top10グラフ

**読み**: 05_売上管理, 03_商品マスタ参照(D列大分類) ／ **書き**: なし

## 07 🛒 推奨発注リスト
**機能**: 04のM列(推奨発注数)>0 のSKU抽出 → 月間上限まで「✅発注推奨/⚠️次回繰越」判定
- ソート: 🔴危険→🟠要発注→🟡注意→🟢余裕→🟣過剰（在庫日数昇順）
- CSVダウンロード ＋「📤 スプシに書き戻し」

**読み**: 04_在庫管理, 03_商品マスタ参照(H列原価), 17_終売SKU ／ **書き**: 11_推奨発注リスト

⚠ 終売SKUは自動除外。`08_自動実行/morning_batch.py` の朝バッチもこの計算ロジックを使う

## 08 📦 FBA補充プラン
**機能**: 自社経由ルートのFBA補充推奨数を計算
- 補充サイクル/安全在庫日数/Amazon販売比率(35%)/段ボール1箱kg をスライダー調整可
- FBA目標 = 販売速度 × Amazon比率 × (cycle+safety)
- 「確定数」列を編集後「📤 スプシに書き戻し」

**読み**: 04_在庫管理(D/F/R), 03_商品マスタ参照(AA列重量), 17_終売SKU ／ **書き**: 12_FBA補充プラン

⚠ 自社経由ルートのみ対象。FBA直送・終売SKUは除外

# ✏️ 編集・入力

## 10 ➕ 新規SKU登録
**機能**: マスタ+04の新規登録 + 各種SKU整理
1. 1件新規登録（マスタ+04同時）
2. マスタにあって04にないSKUを一覧表示→一括追加
3. コバリ子SKU（マスタU列に親SKU参照あり）を04から一括削除
4. 古ASIN行（B0始まり10桁）を新楽天SKUに統合し、新Amazon SKUを書込
5. 旧ASIN行のFBA専売置換（A列+AF列を新FBA SKUに）

**読み**: 03_商品マスタ参照, 04_在庫管理 ／ **書き**: 03_商品マスタ参照, 04_在庫管理

⚠ 新規登録時はL/M列に数式自動生成

## 11 🎯 PUSH対象
**機能**: GAS自動PUSH対象シートの編集
- A: 商品コード（手入力）/ B: 小分類（GAS自動補完）/ C: 今すぐPUSH
- 行追加・削除可、「💾 スプシに保存」

**読み**: PUSH対象 ／ **書き**: PUSH対象

⚠ チェック0件 = 全SKU対象（GAS仕様）

## 12 🚫 終売SKU
**機能**: 終売SKU管理（推奨発注・FBA補充から自動除外）
- 「➕ 新規追加」フォーム + 行削除可
- A: 商品コード / B: 小分類（GAS補完）/ C: 終売日

**読み**: 17_終売SKU ／ **書き**: 17_終売SKU

## 13 🎁 レビュープレゼント
**機能**: 楽天発送CSV取込→04の当月廃棄(J列)に加算→反映済み行を自動削除
- 「📥 CSV一括追加」「➕ 1件追加」「✏️ 編集・削除」タブ
- 列マッピング（プリセット保存可）
- 「⚡ 04に反映実行」

**読み**: レビュープレゼント ／ **書き**: レビュープレゼント, 04_在庫管理(J列)

⚠ 反映後、レビュープレゼント行は自動削除（マスタにないSKUのみ残存＆警告）

## 14 📢 広告費入力
**機能**: 月×チャネル別 広告費・アフィリ入力
- C(広告税抜)→D(税込=C×1.1) / E(アフィリ税抜)→F(手数料込=E×1.3) を自動計算
- 月次サマリで参照

**読み**: 広告費入力 ／ **書き**: 広告費入力

## 15 📥 入荷時
**機能**: 06_入荷時 のデータを 04 のI列(当月入荷)に加算
- F列(実際入荷数) → 04のI列に加算
- R列(調整数) → 発注済みから減算
- 「⚡ 04に反映実行」後、06_入荷時 を自動クリア

**読み**: 06_入荷時 ／ **書き**: 04_在庫管理, 06_入荷時（クリア）

## 16 📋 エクセル発注用
**機能**: 10_エクセル発注用 のデータを 04+マスタに反映
- F列(発注数)÷X列(係数) → 04のL列(発注済み)に加算
- マスタT列(最終発注日)を今日に更新
- 「⚡ 04＋マスタに反映実行」後、10_エクセル発注用 を自動クリア

**読み**: 10_エクセル発注用 ／ **書き**: 04_在庫管理(L列), 03_商品マスタ参照(T列), 10_エクセル発注用（クリア）

## 17 🗒️ 棚卸在庫
**機能**: 月1回棚卸の実カウント入力 → スプシ反映
- 空シートに「📋 棚卸シート準備」で04全SKU書出し
- 「📥 CSVエクスポート」で紙計算→「📤 CSVインポート」（全置換 / C列のみ更新）
- 「💾 入力を保存」

**読み**: 14_棚卸在庫シート, 04_在庫管理 ／ **書き**: 14_棚卸在庫シート

# 🔧 設定・補助

## 20 🔐 認証設定
**機能**: 楽天RMS / Amazon SP-API / LINE Messaging API の認証情報管理
- 楽天RMS: Service Secret / License Key / Shop URL
- Amazon SP-API: LWA Client ID/Secret / Refresh Token / Seller ID / Marketplace ID
- LINE: Channel Access Token / User ID
- 「🏪/📦/📱 テスト」で接続確認

**読み**: なし ／ **書き**: `.streamlit/secrets.toml`

⚠ 保存後はStreamlit再起動推奨

## 21 🔄 プール在庫設定
**機能**: 楽天SKU+Amazon SKUペアを「プール在庫」式に切替
- 04のF列: マスタAE列ありなら VLOOKUP楽天SKU、なければ通常計算
- K列(在庫金額): ミラー行は0でカウント防止
- 「⚡ プール在庫式を設定」「🔧 K列ARRAYFORMULAを更新」

**読み**: 03_商品マスタ参照(AE列), 04_在庫管理 ／ **書き**: 04_在庫管理(F列・K列)

## 22 ⚡ 一括実行（雛形）
**機能**: 朝バッチ相当の処理を選択実行（現状はページへの案内）
- チェックボックス: 推奨発注 / FBA補充 / 月次サマリ / 楽天PUSH / Amazon PUSH

⚠ 現状はリンク案内のみ。完全自動化は `08_自動実行/morning_batch.py` (Task Scheduler) が担う

## 23 🔧 SKU統合（運用見送り中）
**機能**: マスタ複数SKU → 統合SKU(U000001…)に1商品1行化
- 5フェーズ: プレフィックス分析→グルーピング→レビュー→マージ→エクスポート→追従

⚠ 現状運用見送り（手動レビュー大変、楽天SKUベース運用継続）。バックアップ必須
